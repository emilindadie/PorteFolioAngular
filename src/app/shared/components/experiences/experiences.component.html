Context

ITNOVEM, filiale du groupe E-SNCF est spécialisée dans les objets connectés et intervient sur des projets digitaux dans le but d’accélérer l’industrialisation des projets numériques du groupe grâce à ses multiples expertises telles que l'IoT, le Big data, l’agilité.

Missions

  • Développement d’une application web avec Angular en front et NodeJS en back destinée aux personnels de la SNCF afin de surveiller leurs différentes activités à l’aide de différents capteurs installés. Les capteurs envoient des données (états, températures mesurées, température de rail…) à notre plateforme ce qui lui permet de générer des alertes ou sms au responsable de l’activité si nécessaire.
  • Développement d’une seconde application dans le but de simuler l’injection de données afin d’observer le comportement de la première application.
  • Architecture 3 tiers implémentée (la couche présentation a été développé avec le framework Angular, la couche métier avec la plateforme NodeJs ainsi que le Framework NestJs afin de gérer le routage).
  • Réalisation de tests fonctionnels avec Karma pour le FrontEnd, réalisation de tests unitaires avec Chai et Mocha pour le backEnd.
  • Utilisation de Git ainsi que GitLab afin de gérer les différentes versions.
  • Intégration continue avec Jenkins.
  • Utilisation d’une base de données NoSQL (CouchDB) et environnement de production (Microsoft azure)
  • Application de la méthode agile (SCRUM)
  • Utilisation de l’outil JIRA pour la gestion de taches.

Context

Missions

  • Développement en équipe d’une application web de gestion d’objectif, architecture de l’application orienté micro-service.
  • Le BackEnd développé en Java avec le Framework Spring et FrontEnd développé avec le Framework Angular
  • Base de données MySQLl, Java pour le backend, et seconde base de données en MongoDB communicant avec un backend en NodeJS. Communication entre le frontend et chaque backend avec une Gateway (passerelle) qui ensuite redirige vers la bonne Api Rest

Environnements

  • SpringBoot, Java, MySQL
  • Angular, Typescript, Angular Material
  • NodeJS, MongoDB, NestJS / ExpressJS
  • Micro-services, Gateway, API Rest
